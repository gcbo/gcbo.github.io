Hook

Hook能在函数调用前/后获得控制权，运行用户自定义代码，能操作函数中的传进的参数，返回值，从而改变程序流的技术，这与程序插桩技术类似。

Hook Windows API

原理

在执行真正的API之前获得控制权。

通过改写[IAT]

Windows中，调用外部库dll的函数一般通过jmp [IAT]实现，重新填写[IAT]的内容使流程发生改变。

  1. 这种方式需要在运行的进程中更改，因为进程运行时会重写[IAT]的内容使之指向正确的API地址。
  2. 无法hook不通过IAT调用的API，如通过动态加载的dll，LoadLibrary –> GetProcessAddress

通过改写[EAT]

某种程度上也是更改[IAT],因为在进程重写[IAT]是通过查询[EAT]实现的。

  1. 需要改写目标API所在的dll

更改代码

dll载入内存中时，找出要hook的API的实际地址(GetProcessAddress)，修改代码。

修改5 byte(inline hook)

把起始代码修改为jmp/call addr,之后再跳转回来。在32位中，addr是4 byte，表示的是下一个指令与用户代码的距离，addr = 目标地址(用户代码地址) - 源地址(jmp指令开始地址) -5 ，5表示jmp addr的长度。

如何跳回

  1. 构造跳回指令
    ecx是第一条指令的开始地址，0x6+0x2是第一条指令以及第二条指令的长度，所以eax是jmp eax的下一条指令与dst的距离，所以可以跳到dst。跳回之前需要恢复jmp addr。

  2. 在dst函数中脱钩，恢复原来的值，调用API,挂钩。

修改 7 byte(hot patch)

API指令特征

mov edi,edi 是API函数入口，这是无意义的指令，hot patch修改的是nop*5,mov edi,edi,共7 byte，而不改变原有的指令。不需要脱钩和挂钩操作。

  1. 将nop*5，改为 jmp addr,addr = dst-src,dst是目标函数地址，src是API入口地址
  2. 将mov edi,edi 改为 0xebf9,即short jmp回第一个nop地址处，jmp -7 ，0xf9为-7的补码

如何在目标进程中实现Hook

以上几种技术除了改写[EAT]，其他两种都需要能够改写目标进程的内存，一般有两种方法，调试法与注入法。

调试

调试器拥有被调试进程的所有权限，可以对其内存更改。

  1. 借用已有调试器(OllyDbg,WinDbg,X64Dbg等)，可以手动改写汇编代码实现，但是繁琐，没有实用价值。借助自动化脚本，自动Hook，强大。
  2. 自己构造调试器。
具体流程
  1. 对目标进程attach，使之成为被调试者，此时调试器获得控制权
  2. 将目标API起始地址的第一个字节修改为0xcc，继续运行让目标进程获得控制权
  3. 当目标进程调用目标API时，将会中断到调试器，调试器获得控制权
  4. 调试器获取寄存器(GetThreadContext)内存，更改目标进程的寄存器，恢复0xcc为原值，同时让EIP指向API入口点，继续运行让目标进程获得控制权
  5. 目标进程继续运行,执行真正的API代码
  6. 继续让调试器重复步骤2
